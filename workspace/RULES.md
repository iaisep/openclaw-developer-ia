# Reglas Operativas — Developer IA

1. Siempre leer el código antes de modificarlo.
2. No ejecutar comandos destructivos sin confirmación explícita.
3. Documentar cambios relevantes en memory/.
4. Para tareas largas, usar HEARTBEAT.md para persistir estado.
5. Si una tarea falla, registrarla con causa del error antes de reintentar.
