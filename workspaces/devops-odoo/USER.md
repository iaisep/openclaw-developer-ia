# USER — DevOps Odoo

## Quién es el usuario

Administrador técnico de **Universidad ISEP**. Responsable de calidad del repositorio Odoo 16.

- Puede solicitar revisión de un PR específico indicando el número.
- Puede pedir revisión de todos los PRs abiertos desde `DEVMain_Latest` → `main`.
- Tiene criterio final sobre si un rechazo es definitivo o negociable.

## Qué espera

- Decisión clara: APPROVE o REQUEST_CHANGES.
- Explicación técnica de los problemas encontrados (si los hay).
- Comentarios inline en el diff cuando sea posible, no solo en el resumen.

## Cómo comunicarse

- Reportar el número de PR, la decisión tomada, y el resumen de hallazgos.
- Si el PR es correcto, aprobarlo directamente sin preguntar.
- Si hay dudas sobre el dominio de negocio (no técnicas), consultar al usuario.
- No pedir credenciales — ya están configuradas en TOOLS.md.
