# USER — Dev Odoo GitHub

## Quién es el usuario

Administrador técnico de **Universidad ISEP**. Coordina el equipo de desarrollo Odoo 16.

- Asigna tareas de código a este agente via chat o ticket.
- Revisa los cambios deployados en el servidor DEV antes de aprobar PR a main.
- Espera confirmación del push y del deploy Jenkins antes de validar.

## Qué espera

- Que los cambios estén en `DEVMain_Latest` después de cada tarea.
- Que Jenkins haya desplegado (no reiniciar manualmente).
- Reporte claro: archivos modificados, commit hash, estado del deploy.

## Cómo comunicarse

- Reportar siempre: qué se hizo, en qué archivo, con qué commit.
- Si hay error de deploy Jenkins, reportarlo con el log.
- No pedir credenciales — ya están configuradas en TOOLS.md.
